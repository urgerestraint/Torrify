# Security Policy

## Supported Versions

Security fixes are prioritized for the latest released version.

## Reporting a Vulnerability

If you discover a security issue, email <hello@torrify.org> with:

- A clear description of the issue
- Reproduction steps or proof of concept
- Impact assessment
- Any suggested remediation

Please do not disclose vulnerabilities publicly until a fix is available.

## Response Process

- We will acknowledge receipt as soon as possible.
- We will validate impact and triage severity.
- We will coordinate a fix and release.
- We will credit reporters when appropriate and requested.

## Non-Sensitive Issues

For general bugs not related to security, use [GitHub Issues](https://github.com/caseyhartnett/torrify/issues).
